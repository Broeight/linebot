## 四、各審查面向

### 1. XML 解析穩健性 — 良好
- 切塊 `/<tbTable[^>]*>[\s\S]*?<\/tbTable>/g`、逐欄 `[\s\S]*?` 非貪婪取值，值一律 `trim()`。
- `參考牌價` `Number()` 後 `isNaN || <=0` 防呆；顯示 `toFixed(1)`。
- `牌價生效時間` `slice(0,10)` 取 `YYYY-MM-DD`，並加 `dateStr.length===10` 防呆。
- 來源若改 XML 結構 → 解析不到 → 回 `{ok:false}` → 友善訊息（優雅降級，不崩潰）。

### 2. 逾時 / 快取 — 正確
- `AbortController` + `setTimeout(...,8000)` + `finally clearTimeout`，逾時走 catch 回 `{ok:false}`。
- 4h TTL 整包單一 key 快取；命中不對外發 fetch。
- **失敗不污染快取**：僅在 `parsed` 成功時才寫 `cache`；reject/非200/解析失敗都不寫入 → 實測失敗後下次仍會重抓（無負快取毒化）。符合 DESIGN「失敗不污染快取」要求。

### 3. 安全 / 一致性 — 一致
- 無金鑰、CommonJS、繁中使用者訊息、AI 工具回英文摘要（比照 weather/exchangeRate/holiday）。
- XML 正則解析手法與 `invoice.js` 一致；服務分層與 `holiday.js`/`exchangeRate.js` 一致。

### 4. 整合 — 正確
- handler 路由位置：放假區塊（第 143 行）之後、發票對獎（第 163 行）之前，且在 AI fallback（第 213 行）之前 — 符合 PRD「匯率之後、發票前、AI fallback 前」。

- 單一油品 regex `無?鉛?` 正確修正 PRD `[無鉛]?` 字元類筆誤，可吃「95無鉛油價」。
- `tools.js`：def 含 `get_fuel_price`、`product` enum 四值且不在 `required`、`run()` case 補英文 fallback、`timeContext()` 含油價提示 — 全部到位。
