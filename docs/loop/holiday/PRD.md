# PRD：連假／放假查詢功能（台灣行事曆）

**版本** v1.0 | **日期** 2026-06-29 | **作者** PM（Claude）

---

## 一、背景與目標

本 LINE Bot 的家庭使用者橫跨台灣與越南，越南家人在台工作或探親時常需確認台灣是否放假，以安排行程與假期規劃。目前使用者需要離開 LINE 另開瀏覽器查行事曆，流程不順暢。

**目標**：讓台灣與越南家人都能在 LINE Bot 內直接查詢台灣國定假日、補班補假資訊，不需離開 App，越南語家人也能用自然語句查詢。

---

## 二、使用者故事

1. **台灣家人（中文）**：我輸入「今天放假嗎」，Bot 立刻告訴我今天是不是假日、假日名稱，或是補班日。

2. **台灣家人（中文）**：我輸入「下一個連假」，Bot 告訴我幾月幾號到幾號、連放幾天、是什麼假期，方便提前規劃行程。

3. **台灣家人（中文）**：我輸入「7月有哪些假」，Bot 列出該月所有放假日（含補班補假），方便安排月份計畫。

4. **越南家人（越南語）**：我用越南語問「mai có được nghỉ không」（明天放假嗎？），AI 工具查詢後用越南語回覆結果。

5. **越南家人（越南語）**：我問「kỳ nghỉ dài tiếp theo là khi nào」（下一個連假是什麼時候？），AI 工具查詢後用越南語回覆連假區間與名稱。

---

## 三、範圍

### 要做

- 支援中文關鍵字指令查放假（走 `handler.js` 路由）
- 支援自然語句查放假，含越南語（走 `tools.js` AI 工具路由）
- 查詢類型：
  - 今天 / 明天是否放假
  - 下一個連假（連續 ≥ 3 天假期）：日期區間、天數、假期名稱
  - 指定月份的所有假日清單
  - 最近最近的假日（無法明確指定月份時的 fallback）
- 補班日識別：若查詢日是補班日，明確告知「今天是補班日（非假日）」
- 資料範圍：台灣行政院人事行政總處公告之當年度行事曆
- 新服務建立為 `src/services/holiday.js`，對外匯出查詢函式

### 不做（本期）

- 不做越南或其他國家的節假日查詢
- 不做跨年度查詢（同年 12 月查次年資料除外，見邊界風險說明）
- 不做行事曆訂閱 / ics 匯出
- 不做「X 天後是假日嗎」等相對天數計算（「明天」除外）
- 不做農曆節日的農曆日期說明（只顯示西元日期）
- 不做假日倒數計時功能

## 四、功能需求

### 4.1 中文關鍵字指令（handler.js 路由）

| 輸入格式 | 說明 |
|---|---|
| `今天放假嗎` | 查今天是否為假日或補班日 |
| `明天放假嗎` | 查明天是否為假日或補班日 |
| `下一個連假` / `最近連假` | 查從今天起最近一個連續 ≥ 3 天的假期 |
| `最近的假日` / `最近假日` | 查從今天起最近一個放假日（不限連假） |
| `7月假日` / `7月有哪些假` | 查指定月份（1–12）所有假日清單 |
| `放假查詢` / `假日查詢` | 顯示使用說明（子選單提示） |

**關鍵字觸發正則**（供架構師實作參考，不強制）：

- `/^今天放假嗎?$/`
- `/^明天放假嗎?$/`
- `/^(?:下一個|最近|下個)連假$/`
- `/^最近(?:的)?假日$/`
- `/^(\d{1,2})月(?:有哪些假|假日|放假)$/`
- `/^(?:放假|假日)查詢$/`

### 4.2 AI 工具（tools.js 路由，支援任意語言）

新增一個 AI 工具定義 `get_taiwan_holiday`：

- **觸發時機**：使用者用任何語言（尤其越南語）詢問台灣是否放假、連假、假日時間時
- **工具接收參數**：

  | 參數 | 型別 | 必填 | 說明 |
  |---|---|---|---|
  | `query_type` | string（enum） | 是 | `"is_holiday"`（特定日是否放假）/ `"next_long_break"`（下一個連假）/ `"next_holiday"`（最近假日）/ `"month_holidays"`（月份清單） |
  | `date` | string | 否 | 查詢特定日，格式 `"YYYY-MM-DD"`；不填預設今天 |
  | `month` | number | 否 | 查詢月份（1–12），`query_type="month_holidays"` 時使用 |

- **工具回傳**：英文純文字摘要（供 AI 用使用者語言改寫），格式見 4.3
- AI 依使用者語言（中文 or 越南語 or 其他）改寫最終回覆

### 4.3 回覆格式

**今天 / 明天是否放假**

放假日：
```
📅 2026-07-01（三）
🎉 今天是假日：台灣光復紀念日調整放假
```

補班日：
```
📅 2026-06-27（六）
🔧 今天是補班日（非假日）
```

一般工作日（週一–週五）：
```
📅 2026-07-06（一）
✅ 今天是上班日，非假日
```

週六 / 週日（無任何標記）：
```
📅 2026-07-05（日）
😌 今天是週末，不用上班！
```

**下一個連假**

```
🗓 下一個連假：中秋節連假
📆 2026-09-30（三）–10-04（日）
⏱ 共 5 天（含週末）
📍 距今還有 94 天
```

**月份假日清單**

```
📋 2026 年 7 月假日

07-04（六）週末
07-05（日）週末
07-12（六）週末
07-13（日）週末
07-19（六）週末
07-20（日）週末
07-26（六）週末
07-27（日）週末

ℹ️ 本月無國定假日
```

若有國定假日則加入：
```
07-15（三）🎉 XX節放假
```

### 4.4 資料需求

外部資料來源的選型決策留給架構師，但 PRD 要求的**資料欄位**如下（每筆假日記錄）：

| 欄位 | 型別 | 說明 |
|---|---|---|
| `date` | string（`YYYY-MM-DD`）| 日期 |
| `isHoliday` | boolean | 是否為放假日（國定假日、補假、彈性放假） |
| `holidayName` | string | 假日名稱（例如「元旦」「春節」「勞動節補假」），非假日則空字串 |
| `isCompensatory` | boolean | 是否為補班日（工作日但被調成上班） |
| `description` | string | 行政院公告說明（選填，可為空） |

**資料來源選型限制（PM 角度）**：

- **必要條件**：免費、免 API Key（或僅需一次性設定），可取得上述欄位
- **建議方向**：行政院人事行政總處「政府行政機關辦公日曆表」開放資料（data.gov.tw）或 GitHub 上已整理的 JSON 格式台灣行事曆（例如 `ruyut/TaiwanCalendar`），具體選型由架構師決定
- 資料更新頻率：年度公告制，通常每年 11–12 月公告次年行事曆，與當年 1 月進行最終確認
- 資料可離線本地快取（非實時 API），以 JSON 形式存於專案內或 `data/` 目錄

## 五、驗收條件

每條均需可執行，有明確預期輸出。

| # | 測試輸入 | 預期輸出 | 通過標準 |
|---|---|---|---|
| AC-01 | `今天放假嗎`（在一般工作日執行） | 回覆「今天是上班日，非假日」 | 正確識別工作日 |
| AC-02 | `今天放假嗎`（在國定假日執行） | 回覆假日名稱，例如「今天是假日：元旦」 | 正確識別假日並顯示名稱 |
| AC-03 | `今天放假嗎`（在補班日執行） | 明確回覆「今天是補班日（非假日）」 | 補班日不被誤判為假日 |
| AC-04 | `明天放假嗎` | 正確回覆明天的假日狀態 | 日期偏移 +1 天計算正確 |
| AC-05 | `下一個連假` | 回覆含「日期區間、天數、假期名稱」，且連續天數 ≥ 3 | 三項資訊均出現，天數計算正確 |
| AC-06 | `7月假日` | 列出當年 7 月所有假日（含國定假日名稱）及補班日標記 | 清單完整，月份正確 |
| AC-07 | `最近的假日` | 回覆今天之後最近一個放假日（可為週末或國定假日） | 日期 ≥ 今日，且確為假日 |
| AC-08 | 越南語：`mai có được nghỉ không`（明天放假嗎） | AI 呼叫 `get_taiwan_holiday` 工具，用越南語回覆明天假日狀態 | 回覆語言為越南語，含正確假日資訊 |
| AC-09 | 越南語：`kỳ nghỉ dài tiếp theo là khi nào`（下一個連假） | AI 呼叫工具後以越南語回覆連假日期與名稱 | 回覆語言為越南語，資訊正確 |
| AC-10 | `12月假日`（於當年 12 月查詢）| 正確列出 12 月假日 | 不因跨年問題回傳空資料或錯誤 |
| AC-11 | `/help` 或「說明」 | 說明文字中包含放假查詢的使用範例 | help 文字更新，含「今天放假嗎」條目 |
| AC-12 | 查詢時資料檔案不存在或解析失敗 | 回覆「目前無法取得假日資料，請稍後再試」，不崩潰 | 有錯誤處理，無 500 錯誤 |

## 六、邊界與風險

### 6.1 資料來源與維護（中優先）

台灣行事曆資料為年度公告制，每年需手動或自動更新一次。

- **風險**：若工程師使用靜態 JSON 檔，年底需手動更新次年資料；若忘記更新，次年查詢將回傳「找不到資料」
- **緩解**：建立明確的年度更新 SOP，或採用會自動同步的開放資料 API（由架構師決定）

### 6.2 跨年度查詢（低－中優先）

使用者在 12 月底問「最近連假」或「下一個連假」時，答案可能橫跨到次年 1 月（如元旦連假）。

- **風險**：若資料只含當年，查不到跨年連假
- **緩解**：資料快取需同時含當年與次年（若可取得）；若次年資料不存在，回覆「目前尚無 YYYY 年行事曆資料，將於公告後更新」，不回傳錯誤

### 6.3 補班日誤判為假日（高優先）

補班日（週六上班日）是最常被誤判的情況，使用者若誤以為可以放假，影響工作安排。

- **要求**：補班日必須在回覆中明確標示為「補班日」，不得被判為假日

### 6.4 週末與國定假日重疊

若國定假日恰好在週六或週日，通常會有「補假」（移至週一），資料源中應已含補假記錄。

- **要求**：補假日（`isHoliday: true`, `holidayName` 含「補假」字樣）應正確識別並顯示原假日名稱

### 6.5 「連假」定義

連假定義為連續 ≥ 3 個非工作日（包含週末、國定假日、補假，**不含**補班日）。

- **邊界**：「週六週日」本身已是 2 天；若週五或週一另有放假，才構成連假。純粹週末（2 天）不算連假
- **補班日切割**：若補班日夾在假日中間，不視為連假；需切斷計算

### 6.6 查詢月份超出資料範圍

使用者輸入「13月假日」或資料不含該月。

- **要求**：月份 1–12 以外，回覆「月份不正確，請輸入 1 到 12 的數字」；資料不存在，回覆找不到資料的友善訊息

### 6.7 越南語自然語句識別範圍

AI 工具的觸發依賴 Groq 模型理解越南語意圖。

- **風險**：模型可能將「越南的假日」與「台灣的假日」混淆
- **緩解**：AI 工具描述需明確標注「台灣假日」，且工具 description 應說明「僅限查詢台灣（中華民國）的放假資料」
