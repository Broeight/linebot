## 六、已知說明

1. **AC-07 mock 策略**：服務採 30 分鐘記憶體快取，mock fetch 時若目標 base 幣別已被快取，快取路徑會直接返回成功，不會走到 fetch。測試改用首次查詢的 KRW（未快取）驗 fetch 失敗路徑，此行為符合設計預期（快取本身即是可用性保障機制）。
2. **AC-05 語言改寫**：工具層（英文摘要輸出）已驗證正確；模型將摘要改寫為越南語的端到端路徑需 LINE + Groq 整合環境，本次以直接呼叫 `tools.run` 覆蓋工具部分，模型改寫部分屬整合測試範疇，本期以工具層通過為驗收標準（符合 DESIGN §8.3 備注）。
