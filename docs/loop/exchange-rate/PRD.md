# PRD：匯率查詢功能

**版本** v1.0 | **日期** 2026-06-29 | **作者** PM（Claude）

---

## 一、背景與目標

本 LINE Bot 的家庭使用者橫跨台灣與越南，有實際的跨境匯款需求：

- 台灣家人定期匯款給越南家人（TWD → VND）
- 越南家人想確認收到的金額換算是否合理（VND → TWD）
- 目前使用者需要離開 LINE、另開網頁查匯率，來回切換麻煩

**目標**：在現有 LINE Bot 內直接查詢即時匯率，減少切換 App 的摩擦，並讓越南語家人也能用自然語句查詢。

---

## 二、使用者故事

1. **台灣家人（中文）**：我想匯 5000 元台幣給越南，想知道大概能換多少越南盾。我在 LINE 輸入「匯率 TWD VND」或「匯率 台幣 越南盾」，機器人立刻回覆目前匯率，讓我快速估算。

2. **越南家人（越南語）**：我想知道今天台幣兌越南盾的匯率。我用越南語問「tỷ giá TWD VND hôm nay là bao nhiêu？」，機器人呼叫工具後用越南語回覆匯率數字。

3. **換算需求**：台灣家人輸入「5000 台幣換越南盾」，機器人除了顯示匯率，也順便換算出結果（例如：5,000 TWD ≈ 3,870,000 VND）。

---

## 三、範圍

### 要做

- 支援中文關鍵字指令查匯率（走 `handler.js` 路由）
- 支援自然語句查匯率，含越南語（走 `tools.js` AI 工具路由）
- 支援「帶金額換算」：使用者輸入金額，同步算出換算結果
- 支援的幣別（優先）：
  - **TWD**（新台幣）
  - **VND**（越南盾）
  - **USD**（美元）
  - **JPY**（日圓）
  - **CNY**（人民幣）
  - **EUR**（歐元）
  - **KRW**（韓圓）
- 失敗時回覆明確的錯誤訊息（非靜默失敗）
- 新服務建立為 `src/services/exchangeRate.js`，對外匯出查詢函式

### 不做（本期）

- 不做歷史匯率走勢圖
- 不做匯率到價提醒（如「台幣對越南盾到 8.0 通知我」）
- 不做帳戶、銀行轉帳等金流整合
- 不做手動設定「常用幣別」偏好
- 不做超過 7 種幣別的擴充（幣別表固定，不做動態新增）

## 四、功能需求

### 4.1 中文關鍵字指令（handler.js 路由）

| 輸入格式 | 說明 |
|---|---|
| `匯率 TWD VND` | 查兩種幣別的即時匯率 |
| `匯率 台幣 越南盾` | 支援中文幣別名稱（映射至標準代碼）|
| `匯率 USD` | 查 USD 相對 TWD 的匯率（TWD 為預設基準）|
| `5000 台幣換越南盾` | 帶金額換算，顯示匯率並換算結果 |
| `換算 5000 TWD VND` | 另一種帶金額換算格式 |

**關鍵字觸發正則**（實作參考，不限定）：
- `/^(?:匯率|exchange\s*rate)\s+(.+)$/i`
- `/^換算\s+(.+)$/i`
- `/^[\d,]+\s*(?:台幣|TWD|VND|USD|JPY|CNY|EUR|KRW)\s*換\s*.+$/i`

**支援的中文幣別名稱映射**：

| 中文 | 代碼 |
|---|---|
| 台幣、新台幣 | TWD |
| 越南盾、越幣 | VND |
| 美元、美金 | USD |
| 日圓、日幣 | JPY |
| 人民幣 | CNY |
| 歐元 | EUR |
| 韓圓、韓幣 | KRW |

### 4.2 AI 工具（tools.js 路由，支援任意語言）

新增一個 AI 工具定義 `get_exchange_rate`：

- **觸發時機**：使用者用任何語言（尤其越南語）詢問匯率、兌換比、換算金額時
- **工具接收參數**：
  - `from`：來源幣別代碼（string，例如 "TWD"）
  - `to`：目標幣別代碼（string，例如 "VND"）
  - `amount`（選填）：要換算的金額（number）
- **工具回傳**：英文純文字摘要（供 AI 用使用者語言改寫），例如：
  `1 TWD = 787.5 VND. Amount: 5000 TWD = 3,937,500 VND. Source: Frankfurter. Updated: 2026-06-29.`
- AI 依使用者語言（中文 or 越南語 or 其他）改寫最終回覆

### 4.3 回覆格式（中文指令路由的範例）

```
💱 匯率查詢
TWD → VND
1 TWD = 787.5 VND

💡 換算：5,000 TWD ≈ 3,937,500 VND

資料來源：Frankfurter（歐洲央行）
更新時間：2026-06-29
```

- 無金額時省略「換算」那行
- 保留「資料來源」與「更新時間」，便於使用者判斷資料新鮮度

### 4.4 外部 API

**採用 [Frankfurter](https://www.frankfurter.app/)（建議首選）**

- 完全免費、不需 API Key
- 資料來源：歐洲央行（ECB），每工作日更新一次（台灣時間約 16:00–17:00）
- HTTPS，CORS 友好
- 端點範例：`https://api.frankfurter.app/latest?from=TWD&to=VND`
- 回傳 JSON：`{ "amount": 1, "base": "TWD", "rates": { "VND": 787.5 } }`
- **限制**：不含 VND（越南盾），因 ECB 不追蹤。見「邊界與風險」。

**備選方案（僅在 Frankfurter 不足時考慮）**

| API | 特色 | 金鑰 |
|---|---|---|
| [exchangerate.host](https://exchangerate.host) | 含 VND，需免費帳號 | 需要（免費層）|
| [ExchangeRate-API](https://www.exchangerate-api.com/) | 含 VND，免費 1500 次/月 | 需要（免費層）|
| [Wise 公開頁面 scraping] | 不穩定，不建議 | — |

**初期建議**：優先使用 ExchangeRate-API 免費層（含 VND、月額 1,500 次遠超需求），若不想管金鑰則以 Frankfurter 為主但需注意 VND 缺口。

## 五、驗收條件

每條均需可執行、有明確預期輸出。

| # | 測試輸入 | 預期輸出 | 通過標準 |
|---|---|---|---|
| AC-01 | `匯率 TWD VND` | 含匯率數字（如「1 TWD = 787.5 VND」）及資料來源 | 回覆內出現正確幣別與合理數值範圍 |
| AC-02 | `匯率 台幣 越南盾` | 與 AC-01 相同結果 | 中文幣別名稱正確映射至代碼 |
| AC-03 | `匯率 USD` | 顯示 USD → TWD（預設基準 TWD）的匯率 | 出現 USD 與 TWD，數值在合理範圍 |
| AC-04 | `5000 台幣換越南盾` | 顯示匯率並換算（如「5,000 TWD ≈ 3,937,500 VND」）| 換算數字正確（誤差 ±0.1%）|
| AC-05 | 越南語自然問句（如 `tỷ giá TWD VND`） | AI 呼叫 `get_exchange_rate` 工具後，用越南語回覆匯率 | 回覆語言為越南語，含正確匯率數值 |
| AC-06 | `匯率 TWD ZZZ`（不支援幣別）| 明確錯誤訊息（如「不支援幣別 ZZZ」），不崩潰 | 無 500 錯誤，使用者收到提示 |
| AC-07 | 外部 API 暫時無法連線 | 回覆「目前無法取得匯率，請稍後再試」，不崩潰 | 有 try/catch 覆蓋，回傳友善訊息 |
| AC-08 | `匯率 JPY TWD` | 正確顯示日圓對台幣匯率 | 數值在歷史合理區間（如 0.2–0.3 TWD/JPY）|

| AC-10 | `/help` 或「說明」 | 說明文字中包含匯率查詢的使用範例 | help 文字更新，含「匯率」條目 |

## 六、邊界與風險

### 6.1 VND 幣別缺口（高優先）

Frankfurter 的資料來自歐洲央行，**不含 VND**。這是本功能最核心幣別之一。

- **風險**：使用 Frankfurter 時查 TWD/VND 會回傳錯誤或空值
- **緩解**：改用 ExchangeRate-API 免費層（含 VND）；或以 USD 為橋接（TWD→USD→VND）但精度略降
- **決策點**（留給工程師）：確認是否申請 ExchangeRate-API 免費帳號，或接受 Frankfurter + USD 橋接

### 6.2 免費 API 額度

| API | 免費額度 | 預估用量 | 風險 |
|---|---|---|---|
| Frankfurter | 無限制（無帳號）| 不限 | 低（但無 VND）|
| ExchangeRate-API | 1,500 次/月 | 家庭使用約 30–100 次/月 | 低 |

### 6.3 資料更新頻率

ECB / 多數免費 API 的匯率**每日更新一次**（非即時）。

- 回覆中應顯示「更新時間」，讓使用者知道非即時報價
- **不得**宣稱是「即時匯率」；改用「參考匯率」措辭

### 6.4 API 不穩定／逾時

- 所有外部 API 呼叫必須有 timeout（建議 5 秒）
- 連線失敗或回傳格式異常，均回覆友善的錯誤訊息，不可讓 Bot 崩潰

### 6.5 匯率僅供參考

- 與銀行實際買賣匯率有價差，手續費未計入
- 回覆末尾應加一行免責說明（如「以上為參考匯率，實際依各銀行為準」）

### 6.6 幣別代碼大小寫

- 使用者可能輸入 `twd`、`Twd`、`TWD`，需統一 toUpperCase 後比對
